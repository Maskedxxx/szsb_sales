## ТЕКУЩАЯ АРХИТЕКТУРА

### 📁 Структура модуля tool_calling:
```
app/core/tool_calling/
├── __init__.py                 # Экспорт основных классов
├── service.py                  # Универсальный ToolService
├── adapters.py                 # LLM адаптеры (OpenAI, Mock)
├── horeca/                     # HoReCa специфика
│   ├── __init__.py
│   ├── service.py              # HoReCaHandler
│   ├── mappings.py             # Все маппинги HoReCa
│   └── filters.py              # Функции фильтрации
└── tests/
    ├── __init__.py
    └── test_horeca.py           # Простой тест модуля
```

### 🔄 Пайплайн с Tool Calling:
```
Query → Semantic Routing → Reranking → Key Selection → 
**Tool Calling** → Context Preparation → Final Generation → Response
```

### 🛠️ Ключевые компоненты:
- **ToolService** - Универсальный сервис для всех отраслей
- **OpenAIAdapter** - Интеграция с LLM через OpenAI API
- **HoReCaHandler** - Специализированный обработчик для HoReCa
- **MockAdapter** - Для тестирования без реального LLM

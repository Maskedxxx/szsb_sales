# ЗАДАЧИ ПРОЕКТА SZSB_SALES

## ТЕКУЩАЯ ЗАДАЧА: Создание логики выбора tools LLM для отрасли HoReCa

### ОПИСАНИЕ ЗАДАЧИ
Необходимо создать экспериментальную логику выбора и применения инструментов (tools) для LLM, которая будет встроена в пайплайн обработки запросов между этапами `select_relevant_keys` и `generate_final_answer`. Функциональность активируется исключительно для отрасли "01": "horeca".

### СИСТЕМНЫЙ АНАЛИЗ

#### ЦЕЛЬ
Внедрить промежуточный этап Tool Calling в существующий пайплайн для улучшения точности обработки запросов в сфере HoReCa через использование специализированных инструментов LLM.

#### КОНТЕКСТ
- Текущий пайплайн: Query → Semantic Routing → Reranking → Key Selection → Context Preparation → Final Generation → Response
- Целевой пайплайн для HoReCa: Query → Semantic Routing → Reranking → Key Selection → **Tool Calling** → Context Preparation → Final Generation → Response

#### ТРЕБОВАНИЯ
1. **Функциональные требования:**
   - Активация только для subsector_id = "01" (horeca)
   - Интеграция между `select_relevant_keys` и `generate_final_answer`
   - Совместимость с существующим пайплайном
   - Обратная совместимость для других отраслей

2. **Нефункциональные требования:**
   - Логирование всех операций
   - Обработка ошибок с fallback
   - Модульная архитектура
   - Качественная документация кода
